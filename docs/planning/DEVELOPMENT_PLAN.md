# XGBoost 训练可视化工具 - 开发计划

## 项目概述

本项目是一个面向节能团队的 XGBoost 算法训练可视化工具，专注于提供专业级的模型训练过程可视化能力，帮助数据科学团队高效进行模型开发与实验对比。

## 技术架构

- **前端**：React 19 + TypeScript + Tailwind CSS + shadcn/ui
- **后端**：Node.js + Express + TypeScript + tRPC
- **训练服务**：Python + FastAPI + XGBoost
- **数据存储**：PostgreSQL + Redis + 文件存储
- **部署**：Docker 容器化部署

## 开发环境设置

### 本地开发环境

#### 前置条件
- Node.js >= 20.0.0
- pnpm >= 9.0.0
- Docker Desktop
- Python >= 3.9

#### 安装步骤
1. **克隆项目**
   ```bash
   git clone <repository-url>
   cd xgboost-training-visualizer
   ```

2. **安装依赖**
   ```bash
   pnpm install
   ```

3. **启动 Docker 环境**
   ```bash
   pnpm docker:up
   ```

4. **启动开发服务器**
   ```bash
   pnpm dev
   ```

### 开发流程

1. **分支管理**
   - `main`：主分支，用于生产环境
   - `develop`：开发分支，集成所有功能
   - `feature/*`：特性分支，用于开发新功能
   - `bugfix/*`：修复分支，用于修复 bug

2. **代码规范**
   - 使用 TypeScript 进行类型检查
   - 遵循 ESLint 代码规范
   - 使用 Prettier 进行代码格式化

3. **提交规范**
   - 使用 Conventional Commits 规范
   - 提交信息格式：`<type>(<scope>): <description>`
   - 类型包括：feat, fix, docs, style, refactor, test, chore

4. **代码审查**
   - 所有代码必须经过代码审查
   - 至少需要 1 个审查者批准
   - 确保代码质量和功能正确性

## 技术实现要点

### 模型版本管理
- **数据库设计**：
  - 扩展 `models` 表，添加 `version`、`parent_id` 字段
  - 创建 `model_versions` 表，记录版本历史
  - 建立版本之间的关联关系

- **API 设计**：
  - `GET /api/models/{id}/versions`：获取模型的所有版本
  - `POST /api/models/{id}/versions`：创建新版本
  - `GET /api/models/{id}/versions/{version}`：获取特定版本
  - `POST /api/models/{id}/versions/{version}/rollback`：回滚到特定版本

- **前端实现**：
  - 模型版本管理页面，显示版本历史
  - 版本比较功能，对比不同版本的性能
  - 版本标签管理，为重要版本添加标签

### 自动调参功能
- **后端实现**：
  - 实现调参服务，支持网格搜索、随机搜索、贝叶斯优化
  - 利用 Redis 队列管理调参任务
  - 支持并行调参，提高效率

- **前端实现**：
  - 调参配置页面，定义参数搜索空间
  - 调参策略选择，设置调参算法
  - 调参结果可视化，展示不同参数组合的性能

- **集成**：
  - 与现有训练系统集成，复用并发训练能力
  - 调参结果存储到实验管理系统
  - 支持调参任务的暂停、继续和停止

### 数据质量检测
- **后端实现**：
  - 实现数据质量检测服务，计算质量指标
  - 支持缺失值检测、异常值检测、数据分布分析
  - 生成详细的质量报告

- **前端实现**：
  - 数据质量分析页面，可视化展示质量指标
  - 质量建议展示，提供数据预处理建议
  - 质量监控仪表盘，跟踪数据集质量变化

- **集成**：
  - 与数据上传和预处理流程集成
  - 与特征工程系统集成，提供质量改进建议
  - 支持质量报告的导出和分享
