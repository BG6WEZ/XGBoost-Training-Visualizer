# 测试用例：数据集导入与上传

> 版本：v1.0  
> 更新日期：2026-03-25  
> 说明：聚焦数据集导入、文件夹上传和多 CSV 逻辑数据集能力。

---

## 1. 测试对象

1. 数据资产扫描
2. 单文件上传
3. 多文件上传
4. 文件夹上传
5. 多 CSV 逻辑数据集组装
6. schema 探查与列映射

---

## 2. 测试用例

| 编号 | 用例名称 | 验证点 | 预期结果 |
|------|------|------|------|
| TC-DI-001 | 扫描内置数据目录 | 能扫描 `dataset/` 一级数据源目录 | 正确生成数据资产候选列表 |
| TC-DI-002 | 单 CSV 上传 | 上传单个 CSV 文件 | 成功创建一个数据资产 |
| TC-DI-003 | 多 CSV 同时上传 | 同时选择多个 CSV 文件 | 成功接收全部文件并允许组装 |
| TC-DI-004 | 文件夹上传 | 选择一个包含多个 CSV 的文件夹 | 成功识别成员文件并创建候选数据集 |
| TC-DI-005 | 文件夹含非 CSV 文件 | 文件夹中同时包含 txt/png/ipynb | 忽略非数据文件或单独提示，不阻断有效 CSV 导入 |
| TC-DI-006 | 空文件夹上传 | 上传空目录 | 返回可理解错误，不创建数据资产 |
| TC-DI-007 | 文件夹层级扫描 | 文件夹内含子目录和 CSV | 按约定递归或提示，行为一致且可追踪 |
| TC-DI-008 | 多 CSV 组成一个数据集 | 将多个 CSV 绑定为一个逻辑数据集 | 创建 1 个逻辑数据集，成员文件关系正确 |
| TC-DI-009 | 主表 + 天气表 + 元数据表组装 | 指定不同文件角色 | 数据集成员角色被正确保存 |
| TC-DI-010 | 按建筑拆分的多个 CSV 组装 | 多个 `BNxxx_energy.csv` 组成一个数据集 | 系统允许按文件集合登记一个逻辑数据集 |
| TC-DI-011 | 按时间分片的多个 CSV 组装 | `part-001.csv` 等分片组合 | 系统允许按分片顺序或规则组装 |
| TC-DI-012 | 成员文件缺失主表 | 只上传天气表和元数据表 | 阻止进入训练态并提示主表缺失 |
| TC-DI-013 | 成员文件重复绑定 | 同一文件被重复加入同一数据集 | 系统去重或明确报错 |
| TC-DI-014 | 文件名重复但路径不同 | 两个同名 CSV 位于不同目录 | 系统能区分并保留完整路径信息 |
| TC-DI-015 | 上传超大单文件 | 上传超过阈值的大 CSV | 进入大文件策略或提示分片方案 |
| TC-DI-016 | 上传进度展示 | 上传过程查看进度 | 进度、速度、状态显示正确 |
| TC-DI-017 | 断点恢复 | 上传中断后重试 | 能继续上传或重新创建任务且状态清晰 |
| TC-DI-018 | 时间列自动识别 | CSV 中包含标准时间列 | 系统自动识别 `timestamp/date` 等时间列 |
| TC-DI-019 | 时间列手动修正 | 自动识别失败时用户手工选择 | 保存后的 schema profile 正确 |
| TC-DI-020 | 目标列配置 | 用户指定目标列 | 数据资产保存目标列配置 |
| TC-DI-021 | 实体列配置 | 用户指定 building/site/entity 列 | 数据资产保存实体列配置 |
| TC-DI-022 | 多表 Join 键配置 | 指定主表与天气表关联键 | Join 键被正确保存 |
| TC-DI-023 | 样本预览 | 预览已上传成员文件数据 | 每个成员文件都可预览前 N 行 |
| TC-DI-024 | schema 探查 | 分析列类型、缺失率、粒度 | 返回结构化 schema 结果 |
| TC-DI-025 | 错误编码 CSV | 上传编码异常文件 | 给出明确错误或转码提示 |
| TC-DI-026 | 列头缺失 | 上传无列头文件 | 正确报错或进入手工映射 |
| TC-DI-027 | 数据资产持久化 | 上传后刷新页面 | 数据资产与成员文件信息仍存在 |
| TC-DI-028 | 删除数据资产 | 删除逻辑数据集 | 仅删除登记信息/工作区产物，不误删原始源文件 |
