# 测试计划

> 版本：v1.0  
> 更新日期：2026-03-25  
> 说明：用于约束 MVP 阶段的测试范围、测试层次、环境和通过标准。

---

## 1. 测试目标

验证以下核心能力稳定可用：

1. 数据资产扫描与登记。
2. 文件上传、文件夹上传、多 CSV 逻辑数据集导入。
3. schema 探查与列映射。
4. 数据质量检查。
5. 特征工程与切分。
6. XGBoost 训练、监控、结果分析。
7. 实验保存与对比。

---

## 2. 测试范围

### 2.1 包含

1. 前端交互测试
2. API 接口测试
3. worker 任务测试
4. 集成测试
5. 数据导入与训练链路的端到端验证

### 2.2 不包含

1. 生产部署压测
2. 在线预测服务测试
3. 复杂权限测试

---

## 3. 测试层级

### 3.1 单元测试

目标：

- 验证纯函数、schema 识别、切分逻辑、特征生成逻辑。

重点：

- 文件成员解析
- 时间列识别
- Join 键匹配
- 逻辑数据集组装

### 3.2 接口测试

目标：

- 验证数据资产、上传、实验、结果相关 API。

重点：

- 文件上传
- 文件夹上传初始化与完成
- 多 CSV 逻辑数据集创建
- 实验启动和状态查询

### 3.3 集成测试

目标：

- 验证模块之间协作正确。

重点：

- 上传后的数据是否进入资产库
- 资产映射结果是否能供训练使用
- 训练结果是否能被结果页读取

### 3.4 端到端测试

目标：

- 验证真实用户路径。

重点路径：

1. 单 CSV 快速训练
2. 文件夹上传并组成逻辑数据集
3. 多表数据集训练与结果查看

---

## 4. 测试数据策略

### 4.1 基准数据

1. `dataset/building_energy_data_extended.csv`
2. `dataset/HEEW/cleaned data`
3. `dataset/ashrae-gepiii/raw`

### 4.2 专项数据

1. 人工构造的小型多 CSV 数据集
2. 缺失列、重复列、错误编码、错误时间格式等边界数据

### 4.3 上传场景专项样例

1. 单个 CSV
2. 包含 2 个 CSV 的文件夹
3. 包含主表、天气表、元数据表的文件夹
4. 包含非 CSV 杂项文件的文件夹
5. 含重复文件名或空文件的集合

---

## 5. 通过标准

MVP 阶段测试通过需要满足：

1. P0 范围内高优先级测试用例通过率 100%
2. P1 范围内关键链路用例通过率不低于 90%
3. 无阻断级缺陷
4. 无数据资产丢失、成员文件错绑、训练结果错读等严重问题

---

## 6. 缺陷优先级

### P0 阻断

1. 文件夹无法上传
2. 多 CSV 无法组成一个数据集
3. 训练任务无法启动
4. 结果与输入数据集不对应

### P1 高

1. schema 识别错误但可手工修正
2. 质量检查结果不完整
3. 实验对比页部分指标异常

### P2 中

1. UI 文案问题
2. 非关键图表样式问题

---

## 7. 回归重点

每次涉及以下模块变更时必须做回归：

1. 上传模块
2. 数据资产模型
3. schema 探查模块
4. 特征工程模块
5. 训练任务模块

回归最少覆盖：

1. 单文件上传
2. 文件夹上传
3. 多 CSV 逻辑数据集训练
4. 结果查看
