# 测试用例：训练模块

> 版本：v1.0
> 更新日期：2026-03-25
> 说明：聚焦训练配置、任务调度、执行监控和模型保存能力。

---

## 1. 测试对象

1. 实验配置与创建
2. XGBoost 参数验证
3. 训练任务调度
4. 训练进度监控
5. 训练控制（暂停/恢复/停止）
6. 模型保存与加载

---

## 2. 测试用例

| 编号 | 用例名称 | 验证点 | 预期结果 |
|------|------|------|------|
| TC-TR-001 | 创建基础实验 | 输入名称、选择数据集、默认参数 | 成功创建实验，状态为 pending |
| TC-TR-002 | 创建实验时选择子数据集 | 选择已切分的子数据集作为训练数据 | 实验关联正确的子数据集 |
| TC-TR-003 | 配置 XGBoost 基础参数 | 设置 learning_rate, max_depth, n_estimators | 参数正确保存 |
| TC-TR-004 | 配置 XGBoost 高级参数 | 设置 gamma, alpha, lambda, min_child_weight | 参数正确保存 |
| TC-TR-005 | 参数边界验证 - learning_rate | 输入 0 或负数 | 返回验证错误 |
| TC-TR-006 | 参数边界验证 - max_depth | 输入 0 或超过 15 | 返回验证错误 |
| TC-TR-007 | 参数边界验证 - n_estimators | 输入 0 或超过 10000 | 返回验证错误 |
| TC-TR-008 | 使用参数预设模板 | 选择"保守/平衡/激进"模板 | 参数自动填充为预设值 |
| TC-TR-009 | 启动训练任务 | 点击开始训练按钮 | 任务进入队列，状态变为 queued |
| TC-TR-010 | 训练任务排队 | 已有最大并发数训练运行时启动新任务 | 新任务排队等待 |
| TC-TR-011 | 训练开始后状态更新 | 任务从队列中取出开始执行 | 状态变为 running |
| TC-TR-012 | 训练进度实时更新 | 训练过程中查看进度 | 迭代次数、百分比正确显示 |
| TC-TR-013 | 训练指标实时更新 | 训练过程中查看指标曲线 | train_loss, val_loss 正确绘制 |
| TC-TR-014 | 训练日志实时更新 | 训练过程中查看日志 | 日志条目实时追加 |
| TC-TR-015 | 暂停训练 | 点击暂停按钮 | 状态变为 paused，进度保留 |
| TC-TR-016 | 恢复暂停的训练 | 点击恢复按钮 | 状态变为 running，从中断处继续 |
| TC-TR-017 | 停止训练 | 点击停止按钮 | 状态变为 cancelled，释放资源 |
| TC-TR-018 | 训练正常完成 | 训练达到 n_estimators | 状态变为 completed |
| TC-TR-019 | 早停机制触发 | 配置 early_stopping_rounds，验证集指标无改善 | 提前停止训练 |
| TC-TR-020 | 训练失败处理 | 数据格式错误或内存不足 | 状态变为 failed，错误信息回显 |
| TC-TR-021 | GPU 训练（CUDA 可用时） | 选择 GPU 设备训练 | 正确使用 GPU 加速 |
| TC-TR-022 | GPU 不可用时回退 | GPU 显存不足或不可用 | 自动回退到 CPU 训练 |
| TC-TR-023 | 模型文件保存 | 训练完成后 | 模型文件正确保存到 MinIO |
| TC-TR-024 | 模型格式选择 | 选择 JSON/UBJ/Pickle 格式 | 按指定格式保存模型 |
| TC-TR-025 | 模型文件下载 | 下载已保存的模型 | 文件正确下载 |
| TC-TR-026 | 训练超时处理 | 训练超过配置的超时时间 | 自动停止并标记超时 |
| TC-TR-027 | 并发训练限制验证 | 启动超过最大并发数的训练 | 超出部分排队等待 |
| TC-TR-028 | 实验配置保存为模板 | 保存当前配置为可复用模板 | 模板列表中出现新模板 |
| TC-TR-029 | 从模板创建实验 | 选择已保存的模板创建实验 | 参数自动填充为模板值 |
| TC-TR-030 | WebSocket 连接断开重连 | 训练过程中网络断开 | 自动重连并恢复进度显示 |
