## 一、任务背景

对 T43 汇报与仓库状态复核后，确认以下事实：

1. T43 的三项核心修复大体成立：
   - T42 最终结论已改为 `⚠️ 部分完成`
   - T42 证据文件已改为带轮次和时间戳的新文件名
   - `scripts/main-gate.sh` 已不再使用 `grep -oP`，且 API pytest 退出码不再被 `tee` 掩盖
2. 当前工作机确实不能做 Unix 实跑：
   - `bash.exe` 指向 WSL
   - 当前环境执行 `bash --version` 失败，报错 `/bin/bash` 不存在
   - 因此 T43 将 Unix 实跑列为阻断项，这一点口径基本成立
3. 但 T43 仍未完成“全部引用同步”和“声明完全准确”两项要求：
   - T41 汇报仍残留旧证据路径 `docs/tasks/M7/evidence/t42/main-gate-output.md`
   - T42 汇报“修改文件清单”仍写旧证据文件名，而非当前规范文件名
   - 因此 T43 汇报中的“✅ 同步所有引用位置”不成立，属于过度声明

本轮任务不新增功能，不重跑大门禁，不扩展测试范围，只收口“文档残留旧路径”和“汇报声明必须与实际一致”两件事。

## 四、详细交付要求

### 4.1 旧证据路径全量清理

必须检查并修正所有仍引用以下旧路径的位置：

`docs/tasks/M7/evidence/t42/main-gate-output.md`

`docs/tasks/M7/evidence/t42/t42-main-gate-output-20260410-100021.md`

要求：

1. 不允许只改正文、不改表格或文件清单。
2. 不允许只改最新汇报，遗漏历史仍需被当前轮次引用的文档。
3. 若某处保留旧路径只是为了说明“已删除的旧文件”，必须明确写成“旧路径/已删除”，避免被误读为现行证据路径。

### 4.2 T43 声明完整性修正

必须修正 T43 汇报中与事实不一致的表述，尤其是：

1. `✅ 同步所有引用位置` 不能保留，除非实际已完成全量同步。
2. `已完成任务`、`最终结论`、`已完成并已验证` 三处措辞必须彼此一致。
3. 若修正后仍存在未同步项，则结论不得写成暗示“本轮已经彻底收口”的表述。

### 4.3 一致性复核

至少复核以下三类一致性：

1. 证据路径一致性：T41 / T42 / T43 中引用的 T42 门禁证据路径必须一致。
2. 结论边界一致性：`已完成并已验证 / 已完成但未独立验证 / 未完成或不在本轮范围` 三层必须前后一致。
3. 阻断说明一致性：Unix 未实跑、CI 未远端运行、浏览器冒烟阻断等，必须继续诚实表述，不得被本轮文档修订误写成已解决。
