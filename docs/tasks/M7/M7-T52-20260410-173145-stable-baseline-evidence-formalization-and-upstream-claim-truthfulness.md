## 一、任务背景

对 T51 汇报复核后，确认其核心技术方向已正确：

1. 通过排除 `T51|t51` 文件，确实可以得到重复执行一致的稳定基线 `126 / 95`。
2. 这说明“冻结对象集 + 排除当轮文档”的思路是有效的。

但 T51 仍有两类真实性缺口：

1. **上游声明不实**：T51 声称已修改 T48/T49/T50，去除了不稳定“最终态”表述；但 T50 当前文件中仍同时保留 `90 / 63` 与 `98 / 98` 两组历史最终态数字，只是在文末新增了冻结快照声明，并未真正清除冲突表述。
2. **证据文件形式未达标**：T51 证据文件虽然给出了稳定结果 `126 / 95`，也说明了“连续多次执行一致”，但没有像任务单要求那样，把“第一次执行结果”和“第二次执行结果”分别落盘展示。

因此 T51 当前问题已收敛为：

1. 不是稳定基线方法错了；
2. 而是“汇报对上游整改完成度的表述过头”以及“证据文件未按要求显式记录双次复算结果”。

本轮任务不再改统计方法，不再扩大到 T53 及后续轮次规则设计，只处理：

1. 让 T51 的证据文件形式完全符合任务要求；
2. 让 T51 对 T48/T49/T50 的整改完成度描述与当前仓库真实状态严格一致。

## 四、详细交付要求

### 4.1 证据文件必须显式记录两次执行

`t51-search-evidence-20260410-170440.md` 必须至少补齐以下内容：

1. 第一次执行命令与结果
2. 第二次执行命令与结果
3. 明确说明两次执行发生在所有文件保存完成之后
4. 明确说明两次结果完全一致，因此稳定基线成立

要求：

1. 不能只写“连续多次执行一致”这类概括话术。
2. 必须把两次结果作为可复核条目写出来。
3. 结果数值必须与当前稳定基线一致。

### 4.2 T51 汇报必须诚实描述上游整改范围

若 T50 仍保留 `90 / 63` 与 `98 / 98` 两组历史数字，则 T51 不得再写“去除了其中不稳定的最终态表述”。

此时只能使用如下类型表述之一：

1. **已新增冻结快照声明，但历史冲突数字仍保留**；或
2. **已完成彻底清理**，前提是确实把冲突数字完全移除或重写。

### 4.3 上游文件一致性边界

T48/T49/T50 三份汇报至少必须满足：

1. 都明确属于“冻结快照声明”；
2. 不再把历史数字直接表述为“当前仓库真实统计结果”。

若 T50 仍保留历史数字，必须通过上下文或标题明确它们是历史快照，而不是当前稳定基线。

### 4.4 最终结论必须回答的问题

最终结论必须明确回答：

1. T51 哪一部分已经真正完成。
2. T51 原汇报哪一句话说过头了。
3. 证据文件中双次执行结果分别是多少。
4. T48/T49/T50 当前究竟是“已新增冻结快照声明”还是“已彻底清除冲突历史数字”。
