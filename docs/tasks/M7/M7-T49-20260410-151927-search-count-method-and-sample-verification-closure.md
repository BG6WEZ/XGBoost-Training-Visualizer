## 四、详细交付要求

### 4.1 计数方法固定化

必须明确写出本轮采用的唯一计数方法，例如：

1. PowerShell `Get-ChildItem ... | Select-String ... | Measure-Object`

要求：

1. 不能混用多种工具得出不同统计而不解释。
2. 必须说明统计单位是“命中行数”还是“唯一文件数”或其他口径。
3. 汇报中的总数必须与该方法实际输出一致。

### 4.2 样本核验

对于每个关键词，至少提供一小组真实样本，要求包含：

1. 文件路径
2. 行号
3. 命中文本片段
4. 分类结果（当前事实 / 历史示例 / 任务单描述）

样本不要求覆盖全部命中，但必须说明筛选原则。

### 4.3 分类表回绑搜索结果

若提供分类表，必须满足：

1. 每一行都能在真实搜索结果中找到。
2. 不允许出现工具未命中的行号。
3. 若分类表不是全量表，必须明确写“节选样本”而不是写成全量。

### 4.4 结论回绑方法

最终结论必须明确回答：

1. T48 前版为什么错
2. 本版用什么方法修正
3. 现在的统计和分类是否与该方法输出一致
