## 九、风险与限制

1. **WSL2 环境**：Docker Desktop 运行在 WSL2 上，文件 I/O 和网络性能与原生 Linux 存在差异。
2. **Docker 资源限制**：Docker Desktop 默认分配资源有限（WSL2 20 核中的部分），可能影响高并发下的延迟表现。
3. **bcrypt 密码哈希**：`/api/auth/login` 的高延迟主要来自 bcrypt 计算（设计为 CPU 密集型），后续可考虑使用异步线程池或降低开发环境迭代次数。
4. **uvicorn 单进程**：API 服务以单 worker 模式运行，Python GIL 限制了并发处理能力。
5. **本轮任务是基线测试**：目标是**测量当前性能**而非优化，本报告记录的数值为**真实性能基线**，后续可在 Task 5.3 或专门的性能优化任务中改进。
