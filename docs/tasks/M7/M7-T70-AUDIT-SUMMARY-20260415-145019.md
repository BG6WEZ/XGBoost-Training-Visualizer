## 三、不通过项

### 1. 核心阻断：任务单要求的 `alembic upgrade head` / `downgrade base` 未按要求完成实跑验证

M7-T70 任务单明确要求至少执行：

- `alembic upgrade head`
- `alembic downgrade base`
- 再次 `alembic upgrade head`

但报告中实际列出的命令只有：

- `alembic --version`
- `alembic history`
- `pytest tests/ -q --tb=short`

结论：

- 报告没有完成任务单要求的关键迁移验证。

### 2. 我实际复跑 `alembic upgrade head` 当前会失败

我在 `apps/api` 目录下补充最小环境变量后实际执行：

- `$env:JWT_SECRET='test-secret-key-for-audit-only'; ..\..\.venv\Scripts\alembic.exe upgrade head`

结果直接失败，报错为：

- `Value error, DATABASE_URL must be set via environment variable or .env file`

失败发生在：

- `apps/api/alembic/env.py` 导入 `app.config.settings`
- `app.config.Settings` 初始化时未正确拿到 `DATABASE_URL`

结论：

- 当前实现尚不能证明 `alembic upgrade head` 可执行成功。
- 在通过条件中勾选 “upgrade head 可执行成功” 不成立。

### 3. `alembic downgrade base` 完全未被验证

## 四、补充观察

### 1. `database.py` 里仍会在未迁移状态下继续查询 `users`

当前 `init_db()` 即使检测到未迁移，也会继续执行：

- `select(User).where(User.username == "admin")`

如果数据库尚未迁移、`users` 表不存在，这段逻辑很可能在启动时继续报错。  
这与“仅打印警告并提示执行迁移”的目标并不完全一致。

### 2. `database.py` 中存在未使用导入

`from sqlalchemy import inspect, text` 中的 `inspect` 当前未使用。  
这不是验收阻断项，但说明实现仍有清理空间。
