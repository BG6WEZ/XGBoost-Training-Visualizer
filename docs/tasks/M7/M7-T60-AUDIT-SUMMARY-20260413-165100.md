## 五、遗留问题检查

**Q1:** 全页刷新是否存在竞态条件？  
**A1:** 可接受的最小实现。改密成功后先调用 `getMe()` 更新 localStorage，再进行页面刷新，降低了竞态风险。生产环境可在后续优化为局部状态刷新（但需更复杂的逻辑）。

**Q2:** 改密对话框弹出后，如果用户关闭浏览器立即重新打开怎么办？  
**A2:** 仍然会被路由拦截回到改密页面（因为 mustChangePassword=true 仍在服务端持久化）。符合强制改密的设计目标。

**Q3:** 是否需要前端单元测试？  
**A3:** 前端当前无测试框架（Phase-3 Task 3.2 补全），M7-T60 汇报中也未声称有单元测试，仅做了真实链路验证。符合规范。
