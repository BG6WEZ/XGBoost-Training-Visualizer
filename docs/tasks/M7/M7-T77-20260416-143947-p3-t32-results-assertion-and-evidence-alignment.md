## 四、必须完成的最小工作

### 1) 截图证据与报告必须逐项对齐

本轮必须做到二选一，但必须诚实一致：

#### 方案 A：补齐截图

- 让每个需要截图的测试都真实落盘
- 报告中列出的文件名必须全部存在
- 若宣称 20 张，则目录中必须真的有 20 张

#### 方案 B：收缩报告

- 如果不是每个测试都需要截图，就不要写“全部 20 张”
- 报告必须如实写实际截图数量与文件名
- 不得写不存在的截图文件

### 2) 结果流程必须增加真实结果断言

当前不接受仅有：

- 页面标题可见
- `body` 可见
- 页面包含实验名

至少补上以下任一类真实结果断言：

1. 结果页中可识别的指标字段断言
2. 明确的状态字段断言
3. 结果图表/结果区块存在且内容发生变化的断言

要求：

- 断言必须依赖真实 `experimentId` 或真实 `datasetId`
- 关键元素不存在时必须失败

### 3) 对比页必须增加真实状态变化断言

当前不接受仅打开 `/compare` 然后看标题。

至少补上以下动作链：

1. 进入对比页
2. 选中一个或多个可对比项
3. 验证按钮、计数器、勾选状态、对比区域或 URL 参数发生变化

要求：

- 必须有“动作前”和“动作后”的差异性断言
- 不得只是截图

### 4) 报告必须严格以事实为准

本轮报告必须明确区分：

- 已验证通过
- 未验证
- 因环境限制无法验证

禁止：

- 写“所有问题已解决”但正文又承认未验证核心通过条件
- 写“全部截图已生成”但目录不匹配

### 5) 必须真实执行 Playwright，并附本轮原始输出

至少执行：

```bash
cd apps/web
npx playwright test
```

要求：

- 报告中附本轮完整原始输出
- 输出必须对应当前代码
- 若仍有失败，必须诚实记录
