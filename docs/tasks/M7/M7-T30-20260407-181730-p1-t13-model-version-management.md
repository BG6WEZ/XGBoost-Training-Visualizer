## 一、任务背景

P1-S5 的实验标签与筛选能力已经闭环，当前系统已具备：

1. 实验创建、训练、结果查看与实验筛选；
2. 预测分析与 Benchmark 指标统一输出；
3. 标签清洗、标签录入、展示与筛选链路；
4. 但模型产物仍停留在“每次训练一个模型记录”的单次结果模式，缺少模型版本历史、版本比较和版本回滚能力。

P1-T13 的目标是把模型管理从“能保存一个模型”提升到“能追踪模型演进历史、能比较、能回滚、能审计”。

本轮只做模型版本管理闭环，不推进导出增强、认证或 SHAP 分析。

---

## 二、任务目标

1. 训练完成后自动创建模型版本快照。
2. 提供版本列表能力，至少展示版本号、创建时间、关键指标、标签/状态。
3. 提供版本比较能力，至少能比较配置差异和核心指标差异。
4. 提供版本回滚能力，回滚后可追踪“当前激活版本”。
5. 前后端契约一致，版本结构、比较结构、回滚结果结构稳定。
6. 提供 focused 测试和真实链路证据，证明版本创建、比较、回滚可用。

## 四、详细交付要求

### 4.1 版本数据模型与编号规则

必须定义独立、可审计的模型版本结构，至少包含：

1. `version_id`
2. `experiment_id`
3. `version_number`
4. `created_at`
5. `config_snapshot`
6. `metrics_snapshot`
7. `tags`
8. `is_active` 或等价“当前激活版本”标志
9. `source_model_path` 或等价存储引用字段

要求：

1. 版本号必须有明确规则，建议使用 `v1.0.0` 风格；
2. 自动创建版本时不得覆盖旧版本；
3. 快照字段必须与训练时真实结果对应，不得伪造。

### 4.2 自动版本创建

训练完成后必须自动创建版本快照，至少满足：

1. 实验状态进入 `completed` 后创建版本；
2. 版本保存配置快照与指标快照；
3. 新版本默认成为当前激活版本，或明确说明激活规则；
4. 若自动创建失败，必须有可审计错误，不得静默吞掉。

### 4.3 版本列表与详情

至少提供：

1. 某实验的版本列表接口；
2. 版本详情接口；
3. 前端版本列表展示；
4. 当前激活版本可识别；
5. 至少展示 `version_number`、`created_at`、`rmse/r2/mae` 中的核心指标。

### 4.4 版本比较

必须支持选择 2-3 个版本进行比较，至少比较：

1. 核心指标差异：`RMSE`、`R2`、`MAE` 中至少 2 项；
2. 配置差异：如 `learning_rate`、`max_depth`、`n_estimators`；
3. 对比结构稳定，前端可读。

要求：

1. 比较不得只返回原始版本列表，必须有清晰的 diff 结构或对比结构；
2. 若选择版本数非法，必须返回可读错误。

### 4.5 版本回滚

必须提供回滚能力，至少满足：

1. 可将历史版本设为当前激活版本；
2. 回滚动作有可追踪记录；
3. 回滚后列表与详情能反映当前激活版本变化；
4. 不要求本轮真正重新训练，但必须说明“回滚”是切换活动版本引用还是触发真实模型替换。

要求：

1. 语义必须诚实，不能把“仅修改标志位”写成“已恢复完整训练状态”；
2. 回滚后下载/使用链路若未验证，必须明确写未验证。

### 4.6 前端最小交互

至少补齐以下页面或区域：

1. 版本列表；
2. 版本比较入口；
3. 回滚按钮或确认交互；
4. 激活版本状态展示；
5. 空状态/无版本时提示。

要求：

1. 不得只做静态页面；
2. 必须接真实后端数据；
3. 不得破坏现有实验详情、结果页与下载模型链路。

## 五、多角色协同执行要求（强制）

1. `Versioning-Agent`：设计版本数据结构、编号规则与激活语义。
2. `API-Agent`：实现自动建版本、版本列表、版本比较、版本回滚接口。
3. `Worker-Agent`：在训练完成链路中接入自动版本创建。
4. `Frontend-Agent`：实现版本列表、比较、回滚确认与空状态。
5. `QA-Agent`：编写 focused 测试并采集真实链路证据。
6. `Reviewer-Agent`：检查版本语义是否诚实、是否越界、证据是否真实。

允许执行者一人分角色完成，但汇报必须按角色拆分产出与证据。

## 八、Copilot 审核重点

1. 是否只是增加了版本表结构，但训练完成后并未自动建版本。
2. 版本比较是否只是前端拼接，没有真正对比配置和指标差异。
3. 回滚是否只是改了 UI 文案，没有真实激活版本切换。
4. 版本快照是否引用了不存在或不稳定的模型路径。
5. 是否把“标记激活版本”过度表述为“完整恢复训练环境”。

---

## 九、风险提示

1. 若版本结构直接绑定现有 `Model` 表的一对一关系，可能会暴露当前模型存储结构的演进限制。
2. 若回滚语义不清晰，后续下载、部署和导出会出现不一致。
3. 若训练完成自动建版本失败未显式暴露，会造成“训练成功但无版本记录”的审计缺口。
