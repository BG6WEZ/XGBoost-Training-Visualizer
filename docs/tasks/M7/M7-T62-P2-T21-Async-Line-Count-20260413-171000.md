## 一、任务概述

### 1.1 目标

将大文件（>100MB）的行计数操作从**同步阻塞**改为**异步非阻塞**，防止上传时 API 响应超时。

### 1.2 背景

**当前问题：** `apps/api/app/routers/datasets.py` 中 `upload_file()` 端点在保存大文件后执行同步行计数：

```python
# 同步操作，会阻塞 API 线程
row_count = sum(1 for _ in open(file_path))
```

对于 100MB+ 的文件，这可能导致：
- API 响应超时（>30s）
- 其他并发请求阻塞

### 1.3 范围与分工建议

| 角色 | 任务 | 文件范围 |
|------|------|---------|
| **后端开发** | 异步行计数、采样估算、progress tracking | `apps/api/app/routers/datasets.py`<br/>`apps/api/app/services/storage.py` |
| **后端测试** | 超时测试、采样精度验证 | `apps/api/tests/test_datasets.py` |

## 二、核心修改项

### 2.1 修改方案

#### 2.1.1 文件大小分类

```python
# apps/api/app/routers/datasets.py

import aiofiles
import math

async def upload_file(file: UploadFile, ...) -> dict:
    """
    上传文件，根据大小智能选择行计数策略。
    
    策略：
    - 小文件 (<10MB)：完整计数（同步）
    - 中文件 (10-100MB)：完整计数（异步，aiofiles）
    - 大文件 (>100MB)：采样估算（读前 10000 行 + 文件大小推算）
    """
    
    # 1. 保存文件
    file_path = await save_file(file)
    file_size = os.path.getsize(file_path)
    
    # 2. 根据大小选择行计数策略
    if file_size < 10 * 1024 * 1024:  # <10MB
        # 同步完整计数
        row_count = sum(1 for _ in open(file_path))
    elif file_size < 100 * 1024 * 1024:  # 10-100MB
        # 异步完整计数
        row_count = await count_lines_async(file_path)
    else:  # >100MB
        # 采样估算
        row_count = await estimate_line_count(file_path)
    
    # 3. 返回响应
    return {
        "filename": file.filename,
        "file_path": file_path,
        "row_count": row_count,
        "estimated": file_size > 100 * 1024 * 1024  # 标记是否为估算值
    }
```

#### 2.1.2 异步行计数函数

```python
# apps/api/app/services/storage.py 或 app/routers/datasets.py

async def count_lines_async(file_path: str) -> int:
    """
    使用 aiofiles 异步计数文件行数。
    
    优点：不阻塞事件循环
    缺点：仍然需要读整个文件
    """
    try:
        count = 0
        async with aiofiles.open(file_path, 'r', encoding='utf-8') as f:
            async for line in f:
                count += 1
        return count
    except Exception as e:
        logger.error(f"✗ 异步行计数失败: {e}")
        raise
```

#### 2.1.3 采样估算函数

```python
# 采样估算文件行数（用于超大文件）

async def estimate_line_count(file_path: str, sample_lines: int = 10000) -> int:
    """
    通过采样估算文件行数。
    
    策略：
    1. 读前 sample_lines（默认 10000）行，计算平均行长度
    2. 用文件总大小 / 平均行长度 推算行数
    3. 乘以 1.0 的修正系数（保守估计）
    
    精度：±10% 可接受
    
    Args:
        file_path: 文件路径
        sample_lines: 采样行数（默认 10000）
    
    Returns:
        估算的行数
    """
    try:
        file_size = os.path.getsize(file_path)
        bytes_read = 0
        lines_sampled = 0
        
        async with aiofiles.open(file_path, 'r', encoding='utf-8') as f:
            async for line in f:
                bytes_read += len(line.encode('utf-8'))
                lines_sampled += 1
                if lines_sampled >= sample_lines:
                    break
        
        if lines_sampled == 0:
            return 0
        
        avg_line_size = bytes_read / lines_sampled
        estimated_lines = int(file_size / avg_line_size)
        
        logger.info(
            f"估算行数：采样 {lines_sampled} 行，"
            f"平均行大小 {avg_line_size:.2f} bytes，"
            f"文件总大小 {file_size} bytes，"
            f"估算总行数 {estimated_lines}"
        )
        
        return estimated_lines
    except Exception as e:
        logger.error(f"✗ 采样估算失败: {e}")
        # 降级：无法计数时返回 0（前端可在 UI 中显示"计数中"）
        return 0
```

## 五、已知约束与风险

1. **采样精度**
   - 对于数据分布不均的文件，采样估算误差可能达 ±15%
   - 可接受：前端 UI 可标记"估算行数"以降低期望
   - 若需更高精度，可增加 sample_lines 至 50000（代价是更长的初始延迟）

2. **编码处理**
   - 当前假设文件编码为 UTF-8
   - 若遇到 GB2312、GBK 等编码，会抛出异常
   - 建议在异常处理中加入：尝试 GBK 重试（遗留项）

3. **内存消耗**
   - aiofiles 是事件驱动的，不额外占用内存
   - 采样估算读 10000 行后即停止，内存占用固定

4. **并发上传**
   - 异步行计数不占用线程池，多个大文件可并发上传
   - 注意：文件系统 I/O 仍受限制（可通过 OS 配置优化）
