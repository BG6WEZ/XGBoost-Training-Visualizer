### 2) 把最终成功执行作为唯一验收依据写清楚

报告中必须明确写出：

- 哪一次执行是最终验收依据

- 该次执行的总耗时

要求：

- 报告中完整贴出那次成功执行的原始输出
- 不能只写摘要
- 不能只保留首次失败执行日志

### 4) 报告中必须明确保留历史上下文，但不能混淆结论

可以写：

- 首次执行失败，原因是 Worker 未运行

但必须做到：

- 最终结论仅基于成功执行
- 不得让“失败日志”和“最终结论”混在一起造成歧义

### 5) 结果必须与当前事实完全一致

若当前最终成功执行事实为：

- 13 / 13 通过
- exit code `0`
- 总耗时 `9.06s`

则报告必须原样反映，不得再回退成旧状态，也不得增删不实信息。
