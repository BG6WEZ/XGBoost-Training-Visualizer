## 三、不通过项

### 1. `/health` 未严格满足原计划“仅返回 {\"status\": \"ok\"}”的要求

`LAUNCH_DEVELOPMENT_PLAN.md` 对 `Task 4.3` 的原始要求是：

- `/health` — 存活探针（liveness），**仅返回 `{"status": "ok"}`**，不检查依赖

但当前实际实现为：

- `status`
- `service`
- `timestamp`

可见 [health.py](file:///c:/Users/wangd/project/XGBoost%20Training%20Visualizer/apps/api/app/routers/health.py#L15-L25)

结论：

- 当前 `/health` 已不检查依赖，这是对的
- 但**没有严格收敛到原计划要求的最小返回体**

### 2. 报告声称 11 个测试通过，但当前环境无法复现测试执行

报告写明：

- `python -m pytest tests/test_health.py -v`
- `11 passed in 2.91s`

但我在当前环境实际执行：

```bash
cd apps/api
python -m pytest tests/test_health.py -q
```

结果在收集阶段失败，先后报错：

- `ModuleNotFoundError: No module named 'bcrypt'`
- 安装最小缺失依赖后，又报 `ModuleNotFoundError: No module named 'jose'`

结论：

- 当前报告中的测试结果**无法在当前工作环境复现**
- 报告缺少对测试运行环境 / 依赖前提的明确说明

### 3. 当前 Python 环境与项目声明依赖存在明显漂移，报告未交代

我核对到：

- `apps/api/requirements.txt` 声明了 `passlib[bcrypt]`、`python-jose[cryptography]`、`minio`
- 当前环境中实际缺少至少：
  - `passlib`（初始状态）
  - `bcrypt`（初始状态）
  - `python-jose`
  - `minio`

同时当前环境版本还与项目固定版本有漂移，例如：

- `pytest-asyncio` 当前为 `1.3.0`
- 项目要求为 `0.23.4`

结论：

- 当前测试通过结论缺少“在哪个环境 / 哪个依赖集合下执行”的可追溯性
- 不足以作为本轮验收证据
