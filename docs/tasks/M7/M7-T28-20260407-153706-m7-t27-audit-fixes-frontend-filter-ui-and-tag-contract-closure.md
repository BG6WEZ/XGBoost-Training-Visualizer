### 阻断问题 1：前端筛选交互未真正落地

M7-T27 任务单明确要求实验页至少补齐：

1. 标签展示；
2. 标签筛选入口；
3. 日期或排序筛选控件；
4. 清空筛选状态；
5. 空结果提示。

审计结果：

1. `apps/web/src/pages/ExperimentsPage.tsx` 当前仍然只展示实验列表和创建实验表单；
2. 未发现标签展示；
3. 未发现标签筛选输入；
4. 未发现日期筛选或排序控件；
5. 未发现清空筛选状态交互；
6. 当前“暂无实验”只是无数据列表态，不是筛选后的空结果交互闭环。

这说明 M7-T27 的前端部分目前只完成了 API 类型准备，**没有完成任务单要求的真实前端筛选交互**。

### 阻断问题 2：标签输入与治理未闭环

任务单 4.1 明确要求：

1. 标签字段在创建、详情、列表保持一致；
2. 标签值去重；
3. 去空格；
4. 去空字符串，避免脏数据扩散。

审计结果：

1. 后端 `create_experiment` 当前直接写入 `tags=data.tags if data.tags else []`；
2. 后端 `update_experiment` 当前直接写入 `experiment.tags = data.tags`；
3. 未见统一标签清洗函数；
4. 现有测试还把“重复标签允许但不推荐”写成通过路径，这与任务单“必须去重”相冲突。

这说明标签契约虽然已加字段，但**脏数据治理没有完成**。

### 阻断问题 3：筛选维度未完整覆盖任务要求

任务单要求实验列表支持：

1. 标签筛选；
2. 日期范围筛选；
3. 性能指标筛选或排序；
4. 至少 1 组组合条件筛选。

审计结果：

1. 当前后端已支持标签、日期、名称、状态；
2. 未实现 `sort_by`、`sort_order` 等性能指标排序参数；
3. 前端也未提供排序交互；
4. 当前汇报把任务写成“前端筛选界面已完成”，但其“未验证范围”又写明“本次仅完成类型定义和 API 方法，未实现可视化组件”。

这说明汇报结论与实际实现**自相矛盾**，且任务要求中的“日期或性能指标维度”没有完整闭环。

### 阻断问题 4：汇报把未完成项写成已完成

M7-T27 汇报当前存在以下不一致：

1. 标题区与任务目标表写“前端筛选界面已完成”；
2. 但第八部分又明确写“前端标签展示组件，本次仅完成类型定义和 API 方法，未实现可视化组件”；
3. 风险部分再次承认“前端展示：本次仅完成类型定义，未实现标签展示组件”。

这违反了工作规则中的诚实原则和文档一致性原则，因此当前汇报不能作为通过依据。

## 四、必须完成的修复项

### 4.1 补齐真实前端筛选交互

实验页至少必须补齐以下内容，并接真实数据，不接受静态 UI：

1. 标签展示；
2. 标签筛选入口；
3. 日期范围筛选或排序控件；
4. 清空筛选状态；
5. 筛选后空结果提示；
6. 筛选状态变化后真实触发列表请求或真实更新数据结果。

最低验收要求：

1. 页面上可见筛选入口；
2. 用户能切换条件并看到结果变化；
3. 不得破坏当前实验创建和列表查看链路。

### 4.2 补齐标签清洗与契约治理

必须把标签清洗从“字段存在”提升到“契约稳定”：

1. 去掉空字符串；
2. 去掉首尾空格；
3. 去重；
4. 创建、更新、详情、列表四条链路行为一致；
5. 汇报与测试不能再把“重复标签保留”写成通过标准。

### 4.3 补齐额外维度筛选或排序闭环

任务单要求“日期范围和性能指标进行筛选或排序”。本轮至少需完成其中一种完整闭环，并在前后端一致落地：

可接受方案二选一：

1. **日期范围闭环方案**：前端提供开始/结束时间筛选控件，真实调用 `created_after` / `created_before`；
2. **性能排序闭环方案**：后端增加 `sort_by` / `sort_order`，支持至少一种性能指标排序，前端提供排序控件。

若选择仅完成日期范围方案，也必须在汇报中明确：本轮未实现性能指标排序，但已满足任务单中的“日期范围”维度要求。

### 4.4 修正汇报结论与证据一致性

`M7-T27-R` 必须重写为单一正式报告，要求：

1. 只写真实已实现内容；
2. 不得再把“仅完成类型定义/API 方法”写成“前端筛选界面已完成”；
3. 必须明确已验证和未验证边界；
4. 所有“已执行命令”必须附真实结果。
