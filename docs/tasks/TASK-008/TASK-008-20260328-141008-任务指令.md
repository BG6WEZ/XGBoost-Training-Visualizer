### 任务 1：结果接口契约测试与异常场景防回归（必须完成）

目标：
- 为 `/api/results` 相关接口建立可回归、可审计的契约测试，防止字段漂移与行为回退。

必须完成：
1. 为以下接口新增或补强测试：
- `GET /api/results/{id}`
- `GET /api/results/{id}/download-model`

2. 覆盖至少以下场景：
- 成功读取已完成实验结果（200）
- 无效 experiment_id 格式（400）
- 不存在 experiment_id（404）
- 模型记录存在但文件缺失场景（404 或明确定义的错误码）

3. 契约断言要求：
- 关键字段存在且类型正确（如 `experiment_id`、`status`、`model.object_key`）
- 错误响应 `detail` 文案与状态码稳定

验收标准：
- 新增/增强测试可在 `.venv` 中稳定通过
- 关键成功和失败路径均有自动化覆盖
- 报告中提供测试用例与输出证据
