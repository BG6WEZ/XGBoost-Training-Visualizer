## 二、任务目标

### 任务 1：文件夹上传能力实测与证据固化

围绕“目录型资产”执行一轮真实链路：

1. 从 `dataset/` 选择一个目录型数据资产（必须包含多个文件）
2. 执行扫描与登记（资产 -> 数据集）
3. 验证数据集详情中 `file_count > 1`
4. 提供实际接口请求与响应证据

**最小验收点**：
- 有 1 个目录型资产成功登记
- 数据集记录中存在多个成员文件

### 任务 2：多 CSV 逻辑数据集导入实测与证据固化

执行“多 CSV 绑定为一个逻辑数据集”链路：

1. 构造或选择至少 2 个 CSV 作为同一逻辑数据集成员
2. 通过 API 创建该数据集（`files[]` 至少 2 项）
3. 校验列表和详情接口返回一致
4. 验证后续可用于切分（执行一次 split）

**最小验收点**：
- 新建数据集 `file_count >= 2`
- split 成功返回 subset

### 任务 3：MVP 验收证据包汇总

整理 M0-M4 + M1补强的证据索引，生成“验收证据矩阵”：

- 能力项
- 对应接口/脚本
- 对应测试文件
- 对应命令输出（含时间）
- 结论（通过/待补）
