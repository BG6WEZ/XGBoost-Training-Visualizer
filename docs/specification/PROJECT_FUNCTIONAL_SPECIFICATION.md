# XGBoost 训练可视化工具 - 项目功能规划

## 1. 项目概述

### 1.1 项目定位
面向节能团队的 **XGBoost 算法训练可视化工具**，专注于提供专业级的模型训练过程可视化能力，帮助数据科学团队高效进行模型开发与实验对比。

### 1.2 目标用户
- **主要用户**：数据科学家、机器学习工程师
- **次要用户**：能源分析师、节能技术顾问
- **用户特点**：具备高级数据分析能力，熟悉机器学习概念，需要精细的参数控制和深度的训练过程洞察
- **使用场景**：团队内部使用，由管理员统一管理账号

### 1.3 部署与运维
- **部署方式**：Docker 容器化部署
- **用户管理**：简化模式，管理员配置账号，无注册流程
- **数据持久化**：所有实验数据、数据集、模型文件持久化存储

### 1.3 核心价值主张
| 价值 | 描述 |
|------|------|
| **可视化洞察** | 实时监控训练过程，直观理解模型学习动态 |
| **实验管理** | 系统化管理训练实验，支持快速对比和回溯 |
| **专业控制** | 完整的 XGBoost 参数暴露，满足高级调优需求 |
| **高效协作** | Web 应用架构，支持团队共享和远程访问 |

### 1.4 项目边界
**包含**：
- XGBoost 模型训练过程可视化
- 训练结果分析与展示
- 实验对比与管理
- 模型文件导出
- **自动化特征工程**
- **数据集切分与子集管理**
- **迁移学习支持**

**不包含**：
- 模型自动调参（AutoML）
- 在线预测服务
- 多模型支持（如 RandomForest、Neural Networks）

### 2.2 模块详细设计

#### 模块一：数据管理

**目标**：为用户提供便捷的数据导入、预处理和切分能力，支持大规模数据处理和灵活的数据集管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **数据上传** | 支持 CSV、Excel 文件上传，拖拽上传 | P0 |
| **大文件支持** | 单文件最大 1GB，超过 1GB 自动切分为多个子文件 | P0 |
| **多文件上传** | 支持同时上传多个文件，批量处理 | P0 |
| **文件夹上传** | 支持上传整个文件夹，自动识别有效数据文件 | P0 |
| **上传进度** | 实时显示上传进度、速度、剩余时间 | P0 |
| 数据预览 | 显示数据前 N 行，列统计信息 | P0 |
| 数据验证 | 检查缺失值、异常值、数据类型 | P0 |
| 列映射 | 指定目标列、特征列、ID 列等 | P0 |
| **数据集切分** | 按时间/空间维度切分为子数据集 | P0 |
| **子集管理** | 管理、查看、导出子数据集 | P0 |
| 数据分布 | 可视化各列的数据分布（直方图、箱线图） | P1 |
| 数据转换 | 缺失值处理、标准化、编码等基础转换 | P1 |
| 数据合并 | 多文件合并为一个数据集 | P2 |

**数据上传流程**：
```
选择/拖拽文件或文件夹 → 格式验证 → 上传/自动切分 → 数据解析 → 显示预览 → 列映射配置 → 数据验证报告 → 确认使用
```

#### 模块二：训练配置

**目标**：提供完整且专业的 XGBoost 参数配置能力

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 基础配置 | 任务类型、目标列、特征列、测试集比例 | P0 |
| **数据集选择** | 选择主数据集或子数据集进行训练 | P0 |
| **迁移学习配置** | 配置源域和目标域数据 | P0 |
| 核心超参数 | learning_rate, max_depth, n_estimators, subsample 等 | P0 |
| 正则化参数 | alpha, lambda, gamma, min_child_weight | P0 |
| 高级参数 | tree_method, booster, objective, eval_metric 等 | P1 |
| 预设模板 | 快速选择常见参数组合（保守/平衡/激进） | P1 |
| 参数验证 | 范围检查、依赖关系检查、冲突提示 | P1 |
| 早停配置 | early_stopping_rounds, 验证集设置 | P0 |

#### 模块三：自动化特征工程

**目标**：自动生成高质量特征，提升模型性能，减少手动特征工程工作量

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **时间特征提取** | 从时间列自动提取年/月/日/时/分/秒/星期/是否周末等 | P0 |
| **滞后特征** | 自动生成滞后特征（lag features）用于时间序列 | P0 |
| **滚动统计特征** | 滑动窗口均值/最大值/最小值/标准差等 | P0 |
| **交互特征** | 特征交叉、多项式特征自动生成 | P1 |
| **编码转换** | 分类变量自动编码（One-Hot/Label/Target Encoding） | P0 |
| **缺失值处理** | 自动识别并处理缺失值（填充/删除/插值） | P0 |
| **异常值检测** | 自动检测并处理异常值 | P1 |
| **特征选择** | 基于重要性/相关性的自动特征筛选 | P1 |
| **特征归一化** | 自动标准化/归一化处理 | P1 |
| **特征报告** | 生成特征工程报告，展示生成的特征及其贡献 | P1 |

## 6. API 设计概要

### 6.1 认证 API
```
POST   /api/auth/login              # 用户登录
POST   /api/auth/logout             # 退出登录
GET    /api/auth/me                 # 获取当前用户信息
POST   /api/auth/change-password    # 修改密码
```

### 6.2 用户管理 API（管理员）
```
GET    /api/admin/users             # 获取用户列表
POST   /api/admin/users             # 创建用户
PUT    /api/admin/users/:id         # 更新用户信息
DELETE /api/admin/users/:id         # 禁用/删除用户
POST   /api/admin/users/:id/reset-password  # 重置用户密码
```

### 6.3 数据管理 API
```
POST   /api/datasets                # 上传数据集（支持分块上传）
POST   /api/datasets/init-upload    # 初始化上传（获取上传ID）
POST   /api/datasets/chunk          # 上传分块
POST   /api/datasets/complete       # 完成上传
GET    /api/datasets/upload-status/:uploadId  # 查询上传进度
GET    /api/datasets                # 获取数据集列表
GET    /api/datasets/:id            # 获取数据集详情
GET    /api/datasets/:id/preview    # 获取数据预览
DELETE /api/datasets/:id            # 删除数据集

# 子数据集管理
POST   /api/datasets/:id/split      # 创建子数据集（切分）
GET    /api/datasets/:id/subsets    # 获取子数据集列表
GET    /api/subsets/:id             # 获取子数据集详情
DELETE /api/subsets/:id             # 删除子数据集
```

### 6.4 特征工程 API
```
POST   /api/feature-engineering/config      # 创建特征工程配置
GET    /api/feature-engineering/config/:id  # 获取配置详情
PUT    /api/feature-engineering/config/:id  # 更新配置
POST   /api/feature-engineering/preview     # 预览特征工程效果
POST   /api/feature-engineering/apply       # 应用特征工程（生成新数据集）

```

| 功能建议 | 决策 | 理由 |

| 数据版本管理 | ⚠️ 低优先级 | 存储成本高，Phase 8+ 考虑 |
| 模型解释性分析 | ✅ 采纳 | SHAP 已在原设计标记为 P2 |
| 多模型支持 | ❌ 不采纳 | 项目专注 XGBoost 深度优化 |
| 响应式设计 | ✅ 采纳 | UI 规范已定义断点 |
| 数据备份恢复 | ⚠️ 低优先级 | Docker 已有卷备份方案 |
| GPU 加速支持 | ✅ 采纳 | 对大数据训练有显著价值 |
| 数据采样和增量学习 | ⚠️ 低优先级 | 复杂度高，Phase 8+ 考虑 |

- **页面精简**：从 10 个页面精简为 8 个核心页面

- **参数深度**：增强参数配置能力，适应高级用户需求
- **对比功能**：新增实验对比模块，支持多实验分析
